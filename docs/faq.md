# FAQ

## General questions
- **What is the platform?**  
  A multi-vertical platform for ride-sharing and food delivery operations.
- **How does it work?**  
  Clients use mobile/web apps and backend APIs for booking, dispatch, delivery, payments, and support.
- **Who can use it?**  
  Passengers/customers, drivers, restaurant partners, admins, and support agents.
- **What are core requirements?**  
  Verified account, valid payment setup (for customers), and approved onboarding (for drivers/restaurants).

## Technical questions
- **What are API limits?**  
  Global default: 300 requests per 60 seconds per client.
- **Which auth methods are supported?**  
  JWT bearer auth with refresh token flow.
- **Is GraphQL available?**  
  Not currently; this repo exposes REST APIs.
- **How is API versioning handled?**  
  `v1` conventions with migration guidance for breaking changes.

## Support questions
- **How do I report a bug?**  
  Open a GitHub Issue with reproduction steps and expected behavior.
- **How do I request a feature?**  
  Open an issue/discussion with use case and acceptance criteria.
- **Where do I get help?**  
  See `docs/getting-help.md` and `docs/community.md`.
- **How do I contact support?**  
  Use in-app support tickets or configured support channels.
