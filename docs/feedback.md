# Feedback and Improvement

## Feedback channels
- GitHub Issues for bugs and feature requests.
- GitHub Discussions for ideas and product feedback.
- Surveys/interviews for structured roadmap input.

## Roadmap voting
- Use Discussions or labeled issues to collect priority votes.
- Summarize top-voted proposals in release planning cycles.

## Beta program
- Offer early access for selected users/partners.
- Collect API, UX, and reliability feedback before broad rollout.
- Recognize contributors in release notes and community updates.

## Documentation feedback
- Report inaccuracies through PRs or issues.
- Include page path, section, and recommended correction.
- Track and prioritize docs quality improvements each release.
