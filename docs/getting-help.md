# Getting Help

## Support channels
- In-app support ticketing (`/api/support/*`)
- Email support (project-specific support alias)
- Community discussions for peer support
- Chat channels (if enabled by the organization)

## Knowledge base
- API docs: `docs/api/README.md`
- Developer docs: `docs/developer-guide.md`
- User docs: `docs/user-guides.md`
- Troubleshooting: `docs/troubleshooting.md`

## Escalation process
1. Capture request ID, timestamps, and impacted user/order/ride IDs.
2. Attempt known fixes from troubleshooting guidance.
3. Escalate to on-call engineering for production-impacting incidents.
4. Track resolution status and communicate updates clearly.

## Community support
- GitHub Discussions for troubleshooting and implementation questions.
- Shared examples and reusable snippets in docs and issue threads.
