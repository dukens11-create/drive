# Contributing Guide

## Contribution workflow
1. Fork/branch from latest `main`.
2. Implement focused changes.
3. Run relevant tests and validations.
4. Open PR with clear summary and risk notes.

## Code of conduct
- Be respectful, inclusive, and constructive.

## Reporting bugs/features
- Use GitHub issues with reproducible steps and expected behavior.

## PR expectations
- Include test evidence.
- Include docs updates when changing behavior.
- Keep changes scoped and reviewable.
